#学び
------------------------------------------------------------------------------
1. ローカルでの準備 (Git)
まずは自分のPC（ローカル環境）で作業の土台を作ります。

リポジトリの作成 (git init): プロジェクトのフォルダをGitの管理下に置きます。

ファイルの作成・編集: 普段通りコードを書きます。

2. 変更の記録 (Git)
変更内容を履歴として保存します。ここは2段構えになっています。

ステージング (git add): 変更したファイルの中から、保存したいものを選んで「準備エリア」に載せます。

コミット (git commit): 準備エリアにある内容にメッセージを添えて、履歴として確定させます。

3. リモートへ送信 (GitHubへ)
ローカルで溜めた履歴を、インターネット上の共有スペースへアップロードします。

プッシュ (git push): ローカルリポジトリの変更を、GitHub上の「リモートリポジトリ」に反映させます。

4. チームでの確認と統合 (GitHub)
GitHub上では、自分の変更を本番コードに混ぜても良いか、仲間と確認し合います。

プルリクエスト (Pull Request): 「この変更を本体に取り込んでください」と依頼を出します。

マージ (Merge): レビュー（確認）が通ったら、変更をメインの枝（ブランチ）に合体させます。

5. 最新状態の同期 (Git & GitHub)
他の人がマージした最新のコードを、自分のPCに取り込みます。

プル (git pull): GitHub上の最新の状態をダウンロードし、自分のローカル環境を最新に更新します。

#コマンド
Gemini の回答
Gitの操作、最初は覚えることが多くて大変ですよね。よく使うコマンドを「作業の流れ」に沿って整理しました。これさえ押さえておけば、日々の開発の8割以上はカバーできるはずです。

1. 基本の流れ（ローカルでの作業）
まずは自分のPCで変更を記録する際に使うコマンドです。

git init: 新しいレポジトリを作成（初期化）します。

git status: 現在のファイルの状況（変更されたか、ステージングされたか）を確認します。頻繁に使います。

git add <ファイル名>: 変更したファイルを「ステージングエリア」に追加します。

git add . で全ての変更を一括追加できます。

git commit -m "メッセージ": ステージングされた変更をローカルレポジトリに記録します。

2. リモートレポジトリとの同期
GitHubなどのサーバーとやり取りするためのコマンドです。

git clone <URL>: リモートにあるレポジトリを自分のPCにコピーします。

git pull: リモートの最新の状態を取得して、自分のコードに反映（マージ）します。

git push: 自分のローカルでのコミットをリモートに送り込みます。

3. ブランチ操作（並行作業）
新機能の開発やバグ修正を、メインのコードを壊さずに行うための機能です。

git branch: ブランチの一覧を表示します。

git checkout -b <ブランチ名>: 新しいブランチを作成して、そこに切り替えます。

git switch <ブランチ名>: 既存のブランチに切り替えます。

git merge <ブランチ名>: 他のブランチの変更内容を、現在のブランチに取り込みます。

4. 履歴の確認と取り消し
「あれ、何したっけ？」という時や、失敗した時に使います。

git log: コミットの履歴を表示します。--onelineをつけると見やすくなります。

git diff: ファイルのどの行が変更されたか、差分を詳しく表示します。

git restore <ファイル名>: ワークツリーの変更を取り消して元に戻します。

git reset --hard HEAD^: 直前のコミットを完全に無かったことにします（※注意して使ってください）。
